## 1. 项目是什么

**校园书递**是一个 B/S 架构的校园二手教材交易平台：

- **用户端**：Vue 3 单页应用（SPA），学生浏览/发布书籍、购物车下单、私信议价、面交确认
- **管理端**：同一套 Vue 工程下的 `/admin/*` 路由，审核书籍/评论、用户管理、合规处理
- **服务端**：Flask 提供 REST API + WebSocket + 静态资源托管
- **数据层**：SQLite（开发默认）或 MySQL 8（生产推荐）

核心场景：**同校同楼筛选、ISBN 填书、按课找书、求购匹配、站内实时消息、RAG 智能找书**。

## 2. 总体架构

```
浏览器 (Vue 3 SPA)
    │
    │  HTTP  /api/*          → REST JSON API
    │  Cookie               → Flask Session 认证
    │  WebSocket /ws/chat   → 私信实时推送
    ▼
Flask 应用 (app.py)
    ├── routes/        路由层：按业务拆分 HTTP 接口
    ├── services/      业务层：校园/RAG/消息/短信/OSS 等
    ├── models.py      ORM 模型（30+ 张表）
    ├── app_helpers.py 公共工具（通知、书籍 enrich 等）
    └── security.py    密码哈希、登录限流
    │
    ▼
SQLAlchemy ORM ──→ SQLite / MySQL
    │
    ├── Redis（可选）→ 缓存、登录限流
    ├── 阿里云 OSS（可选）→ 图片存储
    └── 通义/DashScope（可选）→ RAG 大模型
```

### 开发 vs 生产

| 模式 | 前端 | 后端 | 访问地址 |
|------|------|------|----------|
| 开发 | Vite dev server `:5173` | Flask `:5000` | 浏览器开 5173，API 经 Vite 代理到 5000 |
| 生产 | `npm run build` → `dist/` | Flask 托管 `dist/` + `/api` | 统一 `:5000` |

Vite 代理配置见 `vite.config.js`：`/api`、`/static`、`/assets`、`/ws` 均转发到 Flask。

## 3. 前端技术栈

### 3.1 核心依赖

| 技术 | 作用 |
|------|------|
| **Vue 3** | 组件化 UI，Composition API（`<script setup>`） |
| **Vue Router 4** | 前端路由，`createWebHistory` 模式 |
| **Pinia** | 全局状态（用户信息 `stores/user.js`、应用状态 `stores/app.js`） |
| **Axios** | HTTP 客户端，封装在 `src/utils/api.js` |
| **Vite 5** | 开发热更新 + 生产打包 |

UI 层使用 **Bootstrap 5 + Font Awesome**，无 Element Plus 等重型组件库。

### 3.2 目录结构（前端）

```
src/
├── main.js              入口：Pinia + Router + 启动时 refreshSession
├── App.vue              根组件（Navbar、Toast、OfflineBanner）
├── router/index.js      路由表 + 守卫（requiresAuth、admin 权限）
├── views/               页面（Home、BooksList、BookDetail、admin/* …）
├── components/          可复用组件（BookCard、CommentSection、IsbnScanner …）
├── composables/         组合式逻辑（useAuth、useToast、useAsync …）
├── stores/              Pinia 状态
└── utils/               api.js、auth.js、campus.js、helpers.js …
```

### 3.3 路由与权限

- **公开页**：首页、书籍列表/详情、登录注册（无需登录即可浏览）
- **需登录**：`meta: { requiresAuth: true }` 的页面（发布、购物车、个人中心等）
- **管理后台**：`/admin/*`，路由守卫检查 `user.is_admin` 或 `role`

路由守卫在 `router/index.js` 的 `beforeEach` 中调用 `refreshSession()`，以**服务端 Session 为准**判断登录态。

### 3.4 API 调用方式

`src/utils/api.js` 创建 Axios 实例：

```javascript
axios.create({
  baseURL: '/api',
  withCredentials: true,  // 携带 Cookie，配合 Flask Session
  timeout: 15000
})
```

- 响应拦截器：401 时清 localStorage 并跳转登录页
- 各业务模块导出 API 对象：`authAPI`、`bookAPI`、`orderAPI`、`messageAPI` …

### 3.5 PWA 与离线

- `public/sw.js` + `manifest.webmanifest`：生产环境注册 Service Worker
- `OfflineBanner.vue` + `offlineStore.js`：网络断开提示

## 4. 后端技术栈

### 4.1 核心依赖

| 技术 | 作用 |
|------|------|
| **Flask 2.3** | Web 框架，路由 + 中间件 + Session |
| **Flask-SQLAlchemy 3** | ORM，映射 `models.py` 中 30+ 模型 |
| **Alembic** | 数据库版本迁移（`alembic/versions/`） |
| **Flask-CORS** | 跨域，开发时允许 5173 带 Cookie 访问 |
| **Flask-Caching** | 接口缓存（Redis 或内存 SimpleCache） |
| **flask-sock** | WebSocket（私信推送） |
| **Werkzeug** | 密码 pbkdf2 哈希、静态文件 |
| **PyMySQL** | MySQL 驱动 |
| **gunicorn / waitress** | 生产 WSGI 服务器 |

### 4.2 应用启动流程（app.py）

1. 加载 `.env` 环境变量
2. 创建 Flask app，配置 `SECRET_KEY`、`SQLALCHEMY_DATABASE_URI`
3. 初始化 `db`、`cache`、CORS
4. `register_all_routes(app, …)` 注册全部 HTTP 路由与扩展模块
5. 注册 `before_request`：同步 admin 标记、保护 `/api/admin/*`
6. 开发模式 `python app.py`；生产 `scripts/serve_production.py`

### 4.3 模块化设计

**routes/** — 按领域拆分的 HTTP 路由：

| 文件 | 职责 |
|------|------|
| `route_auth.py` | 登录/注册/验证码/找回密码 |
| `route_books.py` | 书籍 CRUD、审核、搜索 |
| `route_orders.py` | 订单、面交、评价 |
| `route_cart.py` | 购物车 |
| `route_community.py` | 评论、收藏、求购 |
| `route_messaging.py` | 私信、通知、议价 |
| `route_catalog.py` | 分类、公告 |
| `route_admin*.py` | 管理后台 API |
| `route_spa.py` | 所有 Vue 页面路由 → 返回 `index.html` |

**services/** — 可插拔业务扩展，通过 `register_*` 挂载：

| 模块 | 职责 |
|------|------|
| `messaging.py` | WebSocket `/ws/chat` 实时推送 |
| `rag_service.py` + `rag_features.py` | RAG 检索 + LLM 问答 |
| `campus_features.py` | 学籍认证、校区/楼栋、按课找书 |
| `credit_score.py` | 信用分 |
| `admin_compliance.py` | 举报、拉黑、敏感词、封禁 |
| `sms_aliyun.py` + `phone_code.py` | 短信验证码 |
| `oss_storage.py` + `book_media.py` | 图片上传（本地/OSS） |
| `engineering.py` | OpenAPI 文档、缓存配置 |

这种「路由 + 服务注册」模式让 `app.py` 保持精简，新功能只需新增 `services/xxx.py` 并在 `routes/__init__.py` 注册。

## 5. 数据库

### 5.1 双引擎支持

`database_config.py` 按环境变量选择：

- 默认：`sqlite:///campus_book_delivery.db`
- MySQL：`USE_MYSQL=1` 或 `DATABASE_URL=mysql+pymysql://...`

## 6. 认证与安全

### 6.1 Session + Cookie（权威来源）

```
用户登录 POST /api/login
    → Flask 校验用户名密码
    → session['user_id'] = ...
    → Set-Cookie（HttpOnly Session）
    
前端 refreshSession()
    → GET /api/user/info（带 Cookie）
    → Pinia store 更新用户信息
```

- **前端 localStorage** 仅作展示缓存，**不以 localStorage 判断登录**
- 管理员与用户分入口：`session['login_portal']` 为 `user` 或 `admin`
- `/api/admin/*` 由 `before_request` 统一拦截，非 admin 返回 403

### 6.2 密码存储

`security.py`：

- 新密码：`pbkdf2:sha256`（Werkzeug `generate_password_hash`）
- 兼容旧 MD5：登录成功后自动升级哈希

### 6.3 登录限流

- 5 分钟内同一 IP+用户名最多 10 次失败
- 有 `REDIS_URL` 用 Redis 计数；否则进程内内存字典

## 7. 核心业务流程

### 7.1 发布书籍

```
PublishBook.vue
  → 可选 ISBN 扫码/手输（useIsbnLookup → 外部 ISBN API）
  → 图片压缩（imageCompress.js）→ POST multipart 或 base64
  → POST /api/books
  → Book 入库，status=available
  → 管理员审核通过后前台可见（若开启审核）
```

### 7.2 下单与面交

```
BookDetail → 加购 / 立即购买
  → POST /api/orders（扣库存逻辑在 route_orders）
  → 订单 status: pending → confirmed → completed
  → 买卖双方在 Messages 预约面交点（MeetingAppointment）
  → 确认收货 + Review 评价
```

### 7.3 求购匹配

```
WantedList 发布求购（ISBN/关键词）
  → 新书发布或降价时 find_wanted_matches()
  → 写入 notifications + 可选短信/邮件 outbox
```

### 7.4 私信实时消息

```
Messages.vue 打开会话
  → WebSocket 连接 ws://host/ws/chat
  → 发送 { type: 'subscribe', conv_id: '...' }
  → 对方发消息 → HTTP POST 存库 → push_chat_event() 推给在线订阅者
```

WebSocket 依赖 **flask-sock**，Session Cookie 在握手时校验登录态。

### 7.5 RAG 智能找书

```
AiAssistant.vue 用户提问
  → POST /api/rag/chat
  → rag_service.py:
      1. 将书籍/课程/公告/指南切分为 chunks（chunks.json）
      2. 关键词 + 可选向量 embedding 检索 Top-K
      3. 有 DASHSCOPE_API_KEY → 调用通义 qwen 生成回答
      4. 无 Key → 返回检索摘要（降级模式）
```

配置见 `.env.example` 中 `RAG_ENABLED`、`DASHSCOPE_API_KEY`。

## 8. 缓存与性能

`services/engineering.py`：

- 配置了 `REDIS_URL` → `RedisCache`（多进程/Gunicorn 共享）
- 未配置 → `SimpleCache`（单进程内存，开发够用）

典型缓存：书籍列表、分类、公告等读多写少接口。

静态资源：`/static/*` 设置 `Cache-Control: max-age=31536000`。

## 10. 前后端协作约定

### 10.1 响应格式

```json
{ "status": "success", "data": { ... } }
{ "status": "error", "message": "错误说明" }
```

列表接口常带 `items` / `books` / `total` 等字段，具体见各 route 实现。

### 10.2 SPA 路由回退

生产环境所有前端路径（`/booksList`、`/admin/...`）由 `route_spa.py` 返回 `dist/index.html`，由 Vue Router 客户端接管。Flask 只提供 API 和静态文件，**不做服务端模板渲染**（旧 HTML 模板已移除）。

## 13. 答辩常见问题

**Q：为什么选 Vue + Flask，而不是 Spring Boot / Django？**  
A：Vue 3 组件化适合交互多的 SPA；Flask 轻量、路由清晰，Python 生态便于接 RAG/短信/OSS；课程项目开发效率高。

**Q：前后端如何分离又如何部署在一起？**  
A：开发时 Vite 代理分离；生产 `npm run build` 后 Flask 托管 `dist/`，同域 Cookie Session 无需 JWT 跨域问题。

**Q：为什么用 Session 而不是 JWT？**  
A：同域 B/S、服务端需可撤销登录态；Session 存服务端、Cookie 只带 ID，更安全简单。前端 `withCredentials: true` 即可。

**Q：数据库为什么支持 SQLite 和 MySQL？**  
A：SQLite 零配置便于本地演示；MySQL 适合答辩展示「生产级」部署与 Navicat 管理。

**Q：RAG 是什么？不用大模型能用吗？**  
A：检索增强生成：先从平台书籍/指南中检索相关内容，再 optional 用大模型组织回答。无 API Key 时仍可用关键词检索摘要模式。

**Q：WebSocket 和 HTTP 轮询区别？**  
A：私信场景需要实时性；WebSocket 长连接，服务端 `push_chat_event` 主动推送，比轮询省资源、延迟低。

**Q：如何保证交易安全？**  
A：平台定位为**校内面交**非快递；信用分、举报拉黑、管理员审核、ISBN 黑名单、敏感词过滤等多层合规（`admin_compliance.py`）。

**Q：图片存在哪里？**  
A：默认本地磁盘；可切换阿里云 OSS，适合多机部署时统一存储。
