## 1. 一句话 + 定位（30 秒）

**校园书递**是面向高校的 B/S 二手教材交易平台：Vue 3 前端 + Flask 后端 + SQLite/MySQL，强调**同校面交、ISBN 填书、按课找书、私信议价、管理审核**，可选 RAG 智能找书。

## 3. 推荐演示流程（8～10 分钟）

按顺序操作，每步一句话带过技术点。

| 顺序 | 页面/操作 | 讲解要点 |
|------|-----------|----------|
| ① | 首页 `/` | SPA 单页；公告、按课找书/求购/学期入口 |
| ② | 书籍列表 | 分类、同校/楼栋筛选 → `filter_books_campus` |
| ③ | 书籍详情 | 收藏、议价、加购；相似书推荐 |
| ④ | 发布书籍 | ISBN 扫码/查询填书；待审核 `pending` |
| ⑤ | 管理后台 → 书籍管理 | **审核通过** → `available`；体现权限 `is_admin` |
| ⑥ | 购物车 → 下单 | 同卖家合单；订单状态 `pending→pickup→completed` |
| ⑦ | 站内消息 | WebSocket 实时；面交预约 |
| ⑧ | 求购广场 | 发布求购 → 匹配在售书 → 站内通知 |
| ⑨ | AI 助手（可选） | RAG：检索书籍/指南 + 可选大模型 |
| ⑩ | 账户设置 | 学籍认证（校园邮箱后缀） |

**亮点集中讲 3 个**：① Session + 双入口登录（用户/管理分离）② 校园场景（楼栋/课表/面交点）③ 实时私信 + 订单状态机。

## 4. 架构讲解（1 张图口述）

```
浏览器 Vue3 ──HTTP/Cookie──► Flask routes/
                              ├── services/（RAG、消息、校园…）
                              ├── models.py（30+ 表）
                              └── Session 认证
                                    ▼
                              SQLite / MySQL
```

- **前后端分离开发、生产同域部署**：`vite.config.js` 代理 → 生产 `dist/` 由 Flask 托管  
- **模块化**：`routes/__init__.py` 统一 `register_*`  
- **数据完整性**：Alembic + `fk_migration.py` 外键约束

## 5. 技术栈速表（被问到时）

| 层次 | 选型 | 理由 |
|------|------|------|
| 前端 | Vue 3 + Vite + Pinia | 组件化、开发效率高 |
| 后端 | Flask + SQLAlchemy | 轻量、路由清晰、易接 Python AI 生态 |
| 认证 | Flask Session + Cookie | 同域 B/S，可服务端撤销 |
| 实时 | Flask-Sock WebSocket | 私信推送 |
| AI | RAG（检索 + 可选 DashScope） | 无 Key 也能关键词检索 |
| 库 | SQLite 演示 / MySQL 生产 | 迁移脚本齐全 |

## 7. 常见问题（精简答法）

**为什么不用 Spring Boot？**  
课程/团队 Python 栈成熟，Flask 够用且 RAG、短信集成简单。

**为什么 Session 不用 JWT？**  
同域部署，Session 更安全、易登出；前端 `withCredentials` 即可。

**书籍为什么要审核？**  
`pending→available`，防盗版/违规；管理员在后台一键通过。

**WebSocket 做什么？**  
私信新消息、已读回执，避免轮询。

**RAG 没 API Key 能用吗？**  
能。`rebuild_index` 建本地索引，无 LLM 时返回检索摘要。

**如何保证交易安全？**  
平台定位**校内面交**；信用分、举报拉黑、敏感词、ISBN 黑名单。

**图片存哪？**  
默认 `static/uploads/`；可切阿里云 OSS。
