# 校园书递 — 项目报告

> 配套文档：[答辩指南](./答辩指南.md) · [技术栈与原理](./技术栈与原理.md) · [核心代码注释索引](./核心代码逐行注释.md) · [数据库设计](./数据库设计.md) · [文档总索引](./README.md)

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | 校园书递（Campus Book Delivery） |
| 项目类型 | 校园二手教材/书籍交易平台 |
| 目标用户 | 在校学生（买家/卖家）、平台管理员 |
| 运行环境 | Windows / Linux；开发 `http://localhost:5173`；本机/生产 Web `http://127.0.0.1:5000`（`0.0.0.0` 可局域网访问） |

**项目定位**：面向高校场景的 B/S 架构二手书交易系统，支持书籍发布与审核、浏览检索、购物车、收藏、评论（待审）、订单与面交确认、评价、**求购匹配**、**站内私信与面交预约**、**换书订单**、**学籍认证/举报拉黑**、**校园差异化能力**（国内 ISBN 填书/扫码、按课找书、学期专场、同校同楼筛选、倒计时降价等）、**RAG 智能找书**（通义/DashScope 可选）及 **Vue 管理后台**（含评论审核、多校配置）。当前试用数据以 **广东白云学院**（西/北校区、楼栋与面交点）为主，强调校园内当面交易、低价转让与正版流转。

#### 信任与安全（已实现）

- **双入口登录**：用户 `/api/login` 与管理员 `/api/admin/login` 分离；Session 标记 `login_portal`（`user` / `admin`）；非 admin 入口建立的会话无法访问后台 API。
- **密码**：`security.py` 使用 pbkdf2；旧 MD5 用户登录后自动升级。
- **学籍认证**：`campus_schools` 配置邮箱后缀，`POST /api/campus/verify` 校验校园邮箱。
- **举报/拉黑**：`reports`、`user_blocks` 表；私信会话过滤拉黑用户。
- **评论审核**：普通用户评论 `audit_status=pending`，管理端审核后展示。
- **短信网关**：优先 **Webhook 模式**（`SMS_PREFER_WEBHOOK=1` + `SMS_WEBHOOK_URL`）；本机运行 `启动短信Webhook.bat`（8080），验证码写入窗口/`logs/sms_webhook.log`，**不发真实短信**；前端 `SmsWebhookModal.vue` 弹窗提示；亦可配置阿里云 SMS（管理后台四项 + `.env` Secret）。
- **封禁/禁言**：`warning` / `mute` / `ban`；全站 `BanNotice.vue` 顶栏；私信/发布/个人中心拦截；账户设置可申诉。

#### 求购匹配与通知

- 匹配规则：ISBN、书名关键词、预算；上架或发布求购时推送 `new_book` 类通知。
- 列表附带 `match_count`；详情返回 `matches` 书单。

#### 议价与订单

- 买家对在售书报价；卖家接受后自动生成订单并将书置为 `sold`，拒绝/取消其余待处理报价。

#### 换书订单

- `POST /api/orders/exchange`：买家提交交换书名与说明，生成 `order_type=exchange` 订单并通知卖家。

## 四、数据库设计

> 完整表结构文档见 **`docs/数据库设计.md`**（30 张表，含 E-R 图、字段说明、DDL 样例）。重新生成：`python scripts/export_db_doc.py`。

### 4.0 数据处理方式

| 环节 | 实现 |
|------|------|
| 持久化 | SQLAlchemy ORM，用户/书籍/订单等一律入库，**不再同步 `database/users.json`** |
| 库选择 | 默认 SQLite；生产可 `USE_MYSQL=1` + `scripts/init_mysql.py` 建库建表 |
| 结构升级 | `upgrade_db.py`（增量字段/种子）+ **Alembic**（`alembic/versions/`） |
| 外键约束 | `models.py` 声明 `ForeignKey`；迁移 `002_add_foreign_keys` 清孤儿后加约束 |
| SQLite 外键 | `database_config.py` 连接时 `PRAGMA foreign_keys=ON` |
| 复杂字段 | `Text` 存 JSON 字符串（如 `books.imgs`），`to_dict()` 时解析 |
| 图片 | 上传落盘 `static/uploads/books/`（或 `USE_OSS=1`），库中存 URL（`book_media.py`） |
| 试用种子 | `seed_baiyunu_trial.py` 写入白云学院面交点/课程/样例书；`seed_baiyun_schools.py` 高校列表 | 首次或重置演示 |
| 校区迁移 | `campus_seed.py` 将「主校区/生活区」映射为西校区，清理 spot1–8 | `upgrade_db` 时调用 |
| RAG 索引 | `data/rag/chunks.json`：书籍+课程+公告+指南；首次提问或管理员 rebuild | 无 Key 时检索摘要模式 |
| 接口形态 | REST `/api/*`，`jsonify({ status, ... })`；写操作 `db.session.commit()` |
| 前端请求 | `axios` + `withCredentials`；401 清缓存并跳转登录 |

### 4.2 数据库升级

```bash
# SQLite（默认）
python scripts/init_db.py   # 或首次建库（本机部署.bat 会自动 bootstrap）
python upgrade_db.py        # 增量字段与种子
alembic upgrade head        # 外键等 Alembic 迁移

# MySQL（Navicat / 本地 3306）
set USE_MYSQL=1
python scripts/init_mysql.py   # 建库 + create_all + upgrade + 默认账号
python upgrade_db.py
alembic upgrade head
```

**迁移分工**：`upgrade_db.py` 负责历史 `ALTER TABLE` 与 JSON/图片迁移；**Alembic** 负责版本化 schema（baseline + 外键）。应用启动 `bootstrap_database()` 时自动执行 `upgrade_db` 与 `alembic upgrade head`（失败仅 WARN，不阻断）。


### 4.4 数据库物理结构（MySQL 8.0）

本项目 ORM 表名与常见文档命名一致为 **`users` / `books`**（非 `user`/`book`）。Navicat 建库可参考 `models.py` 与 `alembic/versions/`。

```sql
CREATE DATABASE IF NOT EXISTS `campus_book_delivery`
  DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE `campus_book_delivery`;
```

**核心表示例**

| 表 | 说明 | 要点 |
|----|------|------|
| `users` | 用户与管理员 | 主键 `id`；`username` 唯一；`is_admin` + `role`；无独立 admin 表 |
